Software License Implications

[cite_start]

A quick-reference guide to understanding the legal obligations of the open-source tools being used. [cite: 340, 341]

    [cite_start]
  • Permissive (MIT, Apache 2.0): Ideal for government use; allows modification and internal use without restriction. [cite: 342]
  • Copyleft (GPL): Internal use is fine. [cite_start]Distribution of modified versions requires source code release. [cite: 342]
    • [cite_start]
  • Strong Copyleft (AGPL): If a modified version is provided as a network service, source code must be released. [cite: 342]
    • [cite_start]
  • Custom (NPSL): Prohibits redistribution within proprietary software without an OEM license. [cite: 342]
